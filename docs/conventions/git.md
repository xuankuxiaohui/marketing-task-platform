# Git 规范

## 基本原则

- 提交前先确认 `git status --short` 和相关 diff。
- 不回滚用户或其他 Agent 的未提交改动，除非任务明确要求。
- 一个提交只表达一个清晰目的。
- 不提交构建产物、缓存、IDE 私有配置、本地密钥。

## Commit Message

格式：

```text
<type>: <summary>
```

常用类型：

| 类型 | 用途 |
|---|---|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `refactor` | 不改变行为的重构 |
| `docs` | 文档 |
| `test` | 测试 |
| `chore` | 工程杂项 |
| `perf` | 性能优化 |
| `ci` | CI 配置 |

## 交付说明

- 总结行为变化，不只罗列文件。
- 写明验证命令和结果。
- 未验证项必须说明原因。
